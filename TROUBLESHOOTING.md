# Troubleshooting GitHub Actions

## Проблема: Устаревшая версия actions/upload-artifact@v3

Если вы видите предупреждение об устаревшей версии, но в коде уже используется v4:

### Решение 1: Очистка кеша GitHub

GitHub может кешировать старые версии workflow. Попробуйте:

1. Перейдите в **Settings** → **Actions** → **Caches**
2. Удалите все кеши
3. Запустите workflow заново

### Решение 2: Проверка всех workflow файлов

Убедитесь, что во всех файлах используется v4:

```bash
# Проверка всех упоминаний v3
grep -r "@v3" .github/workflows/

# Должно быть пусто
```

### Решение 3: Принудительное обновление

Если проблема сохраняется:

1. Удалите все workflow файлы
2. Создайте их заново с актуальными версиями
3. Сделайте commit и push

## Проблема: Процесс завершен с кодом выхода 1

Это означает, что какой-то шаг в workflow завершился с ошибкой.

### Как найти проблему:

1. Откройте **Actions** вкладку
2. Кликните на упавший workflow
3. Разверните каждый шаг и посмотрите логи
4. Найдите строку с ошибкой

### Частые причины:

- **Ошибки в коде**: Проверьте, что код компилируется
- **Проблемы с зависимостями**: Убедитесь, что `package-lock.json` актуален
- **Переменные окружения**: Проверьте, что все нужные переменные установлены
- **Права доступа**: Убедитесь, что workflow имеет нужные permissions

## Проблема: Workflow не запускается

### Проверьте:

1. **Файл находится в правильной папке**: `.github/workflows/`
2. **Синтаксис YAML корректен**: Проверьте отступы и структуру
3. **Триггеры настроены**: `on:` секция правильная
4. **Ветка правильная**: Убедитесь, что push в `main` или `master`

## Проблема: Деплой не работает

### Для GitHub Pages:

1. Убедитесь, что **Pages включен** в Settings
2. Проверьте, что используется **GitHub Actions** как источник
3. Убедитесь, что workflow имеет права `pages: write`
4. Проверьте, что build проходит успешно

## Полезные команды для отладки:

```bash
# Проверить синтаксис YAML локально
yamllint .github/workflows/*.yml

# Проверить все версии actions
grep -r "uses:" .github/workflows/

# Проверить все триггеры
grep -r "on:" .github/workflows/
```
