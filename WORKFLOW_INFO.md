# Информация о GitHub Actions Workflows

## Активные Workflows

### 1. ✅ Deploy to GitHub Pages (Основной)
**Файл**: `.github/workflows/deploy-github-pages.yml`

Этот workflow используется для автоматического деплоя на GitHub Pages.

**Когда запускается:**
- При push в `main` или `master` ветку
- Вручную через "Run workflow"

**Что делает:**
1. Устанавливает зависимости
2. Собирает React приложение
3. Деплоит на GitHub Pages

**Статус**: ✅ Работает корректно

---

### 2. ✅ CI (Continuous Integration)
**Файл**: `.github/workflows/ci.yml`

Этот workflow проверяет код при каждом push.

**Когда запускается:**
- При push в `main` или `master`
- При создании Pull Request

**Что делает:**
1. Запускает MongoDB и Redis сервисы
2. Устанавливает зависимости
3. Собирает клиент
4. Проверяет форматирование кода

**Статус**: ✅ Работает корректно

---

### 3. ⚠️ Deploy Frontend (Альтернативный)
**Файл**: `.github/workflows/deploy.yml`

Этот workflow используется как альтернативный вариант (только по запросу).

**Когда запускается:**
- Только вручную через "Run workflow"

**Что делает:**
1. Устанавливает зависимости
2. Собирает приложение
3. Сохраняет build артефакты

**Статус**: ⚠️ Отключен от автоматического запуска (чтобы избежать конфликтов)

---

## Рекомендации

### Для автоматического деплоя используйте:
- **Deploy to GitHub Pages** - основной workflow для деплоя

### Если нужно вручную собрать проект:
- Используйте **Deploy Frontend** через "Run workflow"

### Для проверки кода:
- **CI** запускается автоматически при каждом push

---

## Устранение проблем

Если workflow падает с ошибкой:

1. **Проверьте логи** в разделе Actions
2. **Убедитесь что все секреты настроены** (если используются)
3. **Проверьте что GitHub Pages включен** в настройках репозитория
4. **Убедитесь что нет конфликтов** между workflow
